# 调度算法笔记

## 1. FCFS（先来先服务）
- **思想**：按到达顺序执行，非抢占。
- **优点**：简单公平。
- **缺点**：平均等待时间长，护航效应。
- **复杂度**：O(n²) 排序。

## 2. SJF（短作业优先）
- **思想**：执行剩余时间最短的进程，非抢占。
- **优点**：平均等待时间最小。
- **缺点**：可能导致长作业饥饿。
- **复杂度**：O(n²) 或使用优先队列。

## 3. 优先级调度
- **思想**：优先级高的先执行，非抢占。
- **优点**：可区分重要任务。
- **缺点**：低优先级可能饥饿。
- **注意**：数字越小优先级越高。

## 4. RR（时间片轮转）
- **思想**：每个进程执行一个时间片，循环队列。
- **优点**：响应快，适合分时系统。
- **缺点**：时间片大小影响性能；上下文切换开销。
- **关键**：时间片通常 10~100ms。

## 对比总结
| 算法 | 抢占 | 吞吐量 | 响应时间 | 饥饿 |
|------|------|--------|----------|------|
| FCFS | 否 | 低 | 高 | 无 |
| SJF | 否 | 高 | 中 | 可能 |
| 优先级 | 否 | 中 | 中 | 可能 |
| RR | 是 | 中 | 低 | 无 |
